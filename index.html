Como Subir seu Projeto para o GitHub

Siga os passos visuais abaixo para versionar seu código e, ao final, veja como publicar seu site na internet de graça!

Passo 1: Criando o repositório

Passo 2: Nome e descrição do repositório

Como vamos expor na internet deve ser público, mas você pode guardar códigos somente para você de forma privada

Passo 3: Clonando o repositório

Clique no botão indicado para copiar o link do repositório.

Passo 4: git clone + link copiado

Em uma pasta de preferencia na pasta do projeto,

Seja no vsCode ou diretamente no terminal, execute o comando:

git clone [link-copiado]

Passo 5: Vai ser criada uma pasta com o nome do repositório

Passo 6: Copie os arquivos do seu projeto para dentro dessa pasta
recém-criada.

Passo 7: Adicionando os arquivos ao repositório

Git add .

Passo 8: "Commit" A forma que identificamos a mudança (versão)

Git commit -m "Mensagem do commit"

Passo 9: Enviando as alterações para o repositório remoto

Git push origin main (nome da branch)

🚀 Como colocar este site no ar com o GitHub Pages

Agora que seus arquivos já estão no repositório do GitHub, você pode transformar esse código em um site público com um link real para colocar no seu portfólio. Siga o passo a passo:


  1. Renomeie este arquivo para index.html: Os servidores da web (incluindo o GitHub Pages) procuram automaticamente por um arquivo chamado index.html para ser a página inicial do site. Se o seu arquivo se chama seuCodigo.html, renomeie-o para index.html no seu editor e envie essa alteração para o GitHub.
  2. Acesse as Configurações (Settings): Na página principal do seu repositório no GitHub, clique na aba Settings, localizada no menu superior direito (ícone de engrenagem).
  3. Abra o menu Pages: No menu lateral esquerdo da página de configurações, role um pouco para baixo e clique na opção Pages.
  4. Escolha a Branch principal: Na seção Build and deployment, certifique-se de que o campo Source está marcado como Deploy from a branch. Logo abaixo, no campo Branch, clique no botão que diz None, mude para main (ou master) e mantenha a pasta como / (root).
  5. Salve as alterações: Clique no botão Save.
  6. Aguarde a publicação: O GitHub levará entre 1 e 3 minutos para compilar o seu site. Quando você atualizar a página do GitHub Pages após esse tempo, um aviso verde aparecerá no topo mostrando o seu link ao vivo, no formato: https://seu-usuario.github.io/nome-do-repositorio/.
